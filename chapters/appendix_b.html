附录B 角色描述

附录 B 角色描述 此附录包含对各种 PRINCE2 角色的职责的描述。为满足不同角色的需求，这些角色可以根据 一分配给人员的职角色有些角色可能兼职担任，而许多角色也可能根据项目需求分担或组合。 项目保证 概述 项目管理委员会成员负责其各自关注领域（业务、用户或供应商）的项目管理委员会委员会负责 确定项目保证的执行方式，确保在角色和职责方面有明确的规定，并确定被指定担任保证角色的 人员彼此之间如何互动。根据是否拥有为项目团队提供支持所需的知识、技能和精力，保证任务 可以由管理委员会成员自己承担，从更广泛的业务中任命，或从外部方选拔。 职责

  • 就问题、风险和例外报告的评估和解决，向项目管理委员会和项目经理提供建议，并对照 商业论证检查其影响。
  • 就项目小组成员的挑选向项目经理提供建议。
  • 就管理方法向项目经理提供建议。
  • 协助项目管理委员会和项目经理审查产品描述。
  • 审查数据管理实践，以确保数据管理按照项管理数字化和数据管理方法执行，从而向项目 管理委员会成员保证数据得到了妥善管理。
  • 审查问题和变更控制实践，以确保这些实践按照项目的问题管理方法执行，从而向项目管 问题员会成员保证问题得到了妥善管理。
  • 审查风险管理实践，以确保这些实践按照项目的风险管理方法执行，从而向项目管理委员 会成员保证风险得到妥善管理。
  • 根据协定的目标及其容许偏差，确保阶段和项目计划的可行性。
  • 确保对商业论证和项目计划的潜在变更执行影响评估。
  • 根据外部事件和项目进展，检查和监视商业论证。
  • 检查项目计划的变更，以识别任何对业务需求或商业论证的影响。
  • 代表业务检查项目财务状况。
  • 检查项目是否符合总体业务目标。
  • 根据项目进展和任何外部事件检查商业论证。
  • 根据协定的容许偏差确认阶段和项目进展。
  • 确认管理方法符合业务政策，并就管理方法的实施向项目管理委员会做出保证。

附录 B 角色描述 项目总监 概述 项目总监由业务任命，是项目的唯一问责制责任人，最终对项目的成功问责。这种问责制项目总监权。项目总监负责确保项目资金，并对项目的商业论证和持续业务理由负责。他们负责 以符合业务战略的方式有效治理项目，包括确保从长远视角考虑诸如环境、社会和经济影响 等方面。 职责

  • 在项目期间对收益问责（除非由业务管理）。
  • 在项目开展过程中对商业论证问责。监督可行商业论证的开发，确保项目与业务目标保持 一致。
  • 在项目的可持续性报告方面对业务问责。确保可持续性风险被识别、评估和控制。
  • 在与业务分享从项目中吸取的经验教训方面问责。
  • 审查并确认项目方法和交付模式的选择，并确保它们与业务的 ESG 目标兼容。
  • 批准项目概述文件、管理方法、项目管理团队结构、项目产品描述、项目启动文件、项目 计划、阶段计划和任何例外计划（在预测超过阶段层面容许偏差时）。
  • 确保高级用户指定的收益具有性价比、与业务目标一致并且可以实现。
  • 确保高级供应商提议的交付方法和成本的性价比。
  • 确认项目的组织设计。任命项目经理（如果业务未完成此任命），并确认对项目管理团队 的任命。向项目投入人员和业务资源。
  • 决定是否需要项目层面变更预算，并指定预算金额。决定是否需要批准变更的被授权的职 权组织。为每个阶段设变更预算差和变更预算。
  • 确保与商业论证相关的风险被识别、评估、控制。就上报的风险作出决策，特别关注持 续业务风险。根据需要将风险上报给业务。为分配的风风险施风险应对（作为风险行动 负责人）。
  • 确定风险预算。
  • 确保项目资金。
  • 确保实现成果的进展与商业视角保持一致。
  • 对项目经理提出的建议做出响应，就上报的问题做出决策，特别关注持续业务理由。
  • 在预测将超出阶段层面容许偏差时，对例外报告作出决策。根据需要将例外上报给业务。
  • 确保项目是可取、可行且可实现的。
  • 确认项目收尾和项目产品验收。通知业务。

附录 B 角色描述 项目经理 概述 项目经理拥有代表项目管理委员会在协定的项目容许偏差和约束内开展项目经理权。项目经理 负责项目的日常管理。 职责

  • 咨询利益相关方，以准备和维护产品描述和管理方法。
  • 在项目的整个生命周期内中咨询利益相关方，以检查是否有任何目标发生了变更。
  • 设计计划。决定如何应用阶段和交付步骤。
  • 设计、审查和更新项目管理团队结构和工作分解结构。准备角色描述。
  • 准备商业论证和管理方法。在每个阶段结束时对其进行评估和更新。
  • 准备项目计划、阶段计划和工作包描述，并根据需要进行更新。
  • 授权工作包，并设置工作包容许偏差。
  • 确保在项目中，项目风险被识别、评估和控制。
  • 确保小组经理实施其小组计划中协定的管理方法。
  • 建立和维护健康的项目生态系统，并确保项目管理团队的福利。
  • 建立和维护项目记录单，在可能的情况下，由项目支持提供协助。
  • 为分配的风险实施纠正行动和风险应对（作为风险行动负责人）。
  • 在预测到要超出工作包层面容许偏差时，指示纠正行动。
  • 对管理方法所需的程序加以管理，在可能的情况下，由项目支持提供协助。
  • 按阶段计划监视进展。审查商业论证持续可行性的问题及风险的影响。
  • 在预测将超出阶段层面或项目层面容许偏差时，为项目管理委员会编制例外报告。
  • 准备例外计划，以实施应对例外报告的决策。
  • 编制经验教训报告、要点报告、例外报告、阶段竣工报告和项目竣工报告。从项目视角继报告负责可持续性报告。
  • 在项目收尾时，评估并报告项目绩效。

附录 B 角色描述 项目支持 概述 项目支持负责提供各种服务，例如行政支持、就项目工具提供建议和指导、计划支持、风险管 理和变更管理。 这一角色的职责由项目经理负责履行，但也可以将其授权给供应商、个人、业务内部的团体 或其他小组成员。这取决于项目的规模项目经理以及项目经理的技能、知识和精力。有些组 织会建立可履行此角色的项目办公室，以支持项目的交付（见项目组合、项目群和项目办公 室项目支持）。项目支持可帮助在整个项目生态系统中进行集成，包括获得反馈以帮助改进工作方 式、演进项目指导和建立社会凝聚力。 职责

  • 管理专业工具以支持数字化和数据管理方法（例如计划、控制和报告工具）。
  • 监管变更控制和问题处理程序。向项目经理报告影响商业论证的任何拟议或实际产品 变更。
  • 协助小组经理和成员应用项目程序。
  • 协助项目经理维护项目记录单。
  • 协助编制项目计划、阶段计划、工作包描述和小组计划。
  • 协助编制、宣传和存储报告（检查点报告、例外报告、经验教训报告、阶段竣工报告和 项目竣工报告）。
  • 协助项目经理准备问题报告。
  • 为项目计划、阶段计划、小组计划和工作包描述提供基线，并确定存储和分发它们的 方法。
  • 为商业论证、项目概述文件、管理方法和项目启动文件提供基线，并确定存储和分发它 们的方法。
  • 贡献专业知识（例如，计划工具）。
  • 为分配的风险实施风险应对（作为风险行动负责人）。
  • 维护项目基线。
  • 为质量控制、风险控制、变更管理活动、商务管理活动和沟通管理活动提供行政支持。
  • 提供就职和离职方面的帮助。
  • 提供利益相关方分析方面的帮助。

附录 B 角色描述 高级供应商 概述 高级供应商代表参与项目产品交付各个方面工作的供高级供应商高级供应商对供应商交付的产 品质量和项目的技术完整性问责。 职责

  • 就供应商团队的人员方面提供建议，例如，特定的健康、安全和福利需求。
  • 协定关键专业产品的产品描述。
  • 协定管理方法（如适用）。
  • 如产品描述和工作包描述中所定义，协定产品开发中采用的质量技术和工具。提供人员和 资源以执行供应商质量活动。
  • 批准项目产品描述（如适用）。
  • 协助项目经理准备项目计划、阶段计划和工作包描述。
  • 对供应商商业论证问责（如适用）。
  • 投入人员和供应商资源。
  • 确认能够在预期成本范围内交付所要求的产品，并且具备可行性。
  • 确保供应商团体人员的适当参与。
  • 确保与供应商有关的风险被识别、评估和控制（如产品交付）。
  • 确保实现成果的进展与供应商视角保持一致。
  • 确保项目计划和阶段计划与供应商视角保持一致。
  • 为分配的风险实施风险应对（作为风险行动负责人）。
  • 对上报的风险做出决策，特别关注保证整套解决方案的完整性。
  • 从供应商视角继续负责可持续性报告。
  • 对项目经理提出的建议做出响应，就上报的问题做出决策，特别关注保证整套解决方案的 完整性。

附录 B 角色描述 高级用户 概述 高级用户代表用户团体，并且对捕捉用户需求的方法以及与商业论证一致的收益规格问责。 高级用户负责：

  • 确保方法获得用户对项目的认同
  • 监视产品是否符合商业论证中的需求
  • 向业务证明商业论证中预测的收益正在逐步实现
  • 根据需求和收益控制变更（无论是由用户、业务还是项目本身引起的变更）
  • 在项目结束后，确保成功地将产品移交给业务、加以采用，并促进 继续实现收益 高级用户必须确保用户团体中的人员作出充分且持续的承诺来支持这些活动，特别是在项目生 命周期结束后还需要作出收益承诺（如实现收益）的情况下。 职责
  • 协定管理方法（如适用）。
  • 批准项目产品描述和专业产品的产品描述。提供用户的质量期望和验收准则。分配人员和 资源来执行用户质量活动和产品验收。
  • 协助项目经理准备项目计划和阶段计划。
  • 对指定所需成果和收益问责（批准商业论证的依据）。从用户视角定义可持续性报告 要求。
  • 投入人员和用户资源。
  • 为利益相关方分析做出贡献。就用户团体的人员方面提供建议，例如，谁是关键影响者。 确保来自用户团体的人员的适当参与。
  • 确保实现成果的进展与用户视角保持一致。
  • 确保项目计划和阶段计划与用户视角保持一致。
  • 确保与用户有关的风险被识别、评估和控制（例如对收益、运营使用和维护的影风险。对 上报的风险做出决策风险别关注保证预期收益。为分配的风险实施风险应对（作为风险行 动负责人）。
  • 对项目经理提出的建议做出响应，就上报的问题做出决策，特别关注保证预期收益。
  • 在收益评审中，提供实际收益与预测收益的比较说明。
  • 对实现收益问责，并确保项目生产出产品，且该产品可交付期望成果，并且这些收益可产 生期望收益。确认预期收益（来源于项目成果）已实现或能够实现。
  • 验收项目产品并对项目产品的性能问责。

附录 B 角色描述 小组经理 概述 小组经理负责在与项目经理协定的容许偏差和约束范围内交付分配给项目经理作。项目经理按 照商务管理方法，以工作包的形式小组经理配给小组经理。 职责

  • 协助项目经理准备和维护产品描述和工作包描述。与项目经理就工作包描述达成协定。
  • 就小组成员在项目中的角色选择提供建议。管理小组成员。确保团队的福利。
  • 汇编小组计划范围内产品的质量记录。在收到请求时，向项目经理报告产品质量状态。
  • 参与风险的识别、评估和控制。为分配的风险实施风险应对（作为风险行动负责人）。
  • 实施纠正行动。
  • 实施小组计划中协定的管理程序。
  • 告知项目支持问题、风险、经验教训、已完成的质量活动和已完成的产品。
  • 管理小组计划范围内的产品的质量控制。
  • 为每个工作包准备进度表。准备小组计划。
  • 将工作包容许偏差的任何预测偏差通知项目经理。
  • 编制检查点报告，并负责向团队提供可持续性报告。
  • 在其小组计划范围内生产与其产品描述一致的产品。
