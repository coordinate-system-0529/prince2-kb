📋 项目准备 SU

确保项目值得做、做得了、有人管

🔑 关键活动

  • 任命项目发起人和项目经理
  • 捕捉项目经验教训
  • 设计和任命项目管理团队
  • 编制项目概述文件
  • 编制启动阶段计划

📥 输入 → 📤 输出

  • 项目任务书 → 项目概述文件
  • 启动阶段计划
  • 经验教训记录
  • 日志（风险/问题/质量）

➡️ 流向

  • → 项目指导：申请授权启动
  • → 项目启动：提供项目概述文件

📋 项目启动 IP

建立项目基线，获得正式授权

🔑 关键活动

  • 制定项目管理策略
  • 编制项目计划
  • 细化商业论证
  • 建立质量管理方法
  • 建立风险管理方法
  • 编制项目启动文档

📥 输入 → 📤 输出

  • 项目概述文件
  • 项目计划
  • 商业论证
  • 项目启动文档 PID
  • 收益评审计划

➡️ 流向

  • → 项目指导：申请授权交付
  • → 阶段控制：提供阶段计划

📋 阶段控制 CS

日常项目管理，确保阶段目标达成

🔑 关键活动

  • 授权工作包
  • 监督工作包进展
  • 接收已完成工作包
  • 审查问题和风险
  • 上报问题和例外
  • 编制亮点报告

📥 输入 → 📤 输出

  • 阶段计划
  • 工作包（授权）
  • 风险登记册
  • 问题登记册
  • 亮点报告
  • 例外报告

➡️ 流向

  • → 产品交付：授权工作包
  • → 项目指导：提交亮点/例外报告
  • → 阶段边界：阶段即将结束时

📋 产品交付 MP

团队执行工作，交付产品

🔑 关键活动

  • 接受工作包
  • 执行工作
  • 交付产品
  • 报告工作包完成

📥 输入 → 📤 输出

  • 工作包
  • 已完成产品
  • 质量记录
  • 经验教训记录

➡️ 流向

  • → 阶段控制：报告完成
  • → 质量实践：提交质量审查

📋 阶段边界 SB

评审阶段成果，规划下一阶段

🔑 关键活动

  • 编制下一阶段计划
  • 更新项目计划
  • 更新商业论证
  • 报告阶段结束
  • 编制例外计划（如需要）

📥 输入 → 📤 输出

  • 当前阶段计划
  • 项目计划（更新）
  • 商业论证（更新）
  • 下一阶段计划
  • 更新的风险登记册
  • 阶段结束报告

➡️ 流向

  • → 项目指导：申请下一阶段授权
  • → 项目收尾：最后阶段结束时

📋 项目收尾 CP

移交产品，总结经验，正式关闭项目

🔑 关键活动

  • 移交产品给运营方
  • 评估项目绩效
  • 总结经验教训
  • 建议项目收尾
  • 通知项目收尾

📥 输入 → 📤 输出

  • 最终产品
  • 商业论证（最终评审）
  • 经验教训报告
  • 项目收尾通知
  • 收益评审计划（更新）

➡️ 流向

  • → 项目指导：建议收尾
  • → 运营方：移交产品
  • → 组织：经验教训沉淀

📊 核心管理产品流转

管理产品 创建 主要更新 最终确认
商业论证 项目准备 SU → 项目启动 IP、阶段边界 SB → 项目收尾 CP
项目计划 项目启动 IP → 阶段边界 SB → 项目收尾 CP
风险登记册 项目准备 SU → 阶段控制 CS、阶段边界 SB → 项目收尾 CP
问题登记册 项目准备 SU → 阶段控制 CS → 项目收尾 CP
质量登记册 项目启动 IP → 产品交付 MP → 项目收尾 CP
经验教训记录 项目准备 SU → 所有流程持续更新 → 项目收尾 CP
工作包 阶段控制 CS → 产品交付 MP → 阶段控制 CS
亮点报告 阶段控制 CS → — → 项目指导 DP
