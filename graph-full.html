🏛️ 项目指导 DP

项目委员会的决策中枢 — 贯穿全程

🔑 关键活动

  • 授权项目启动（收到SU请求后）
  • 授权项目交付（批准PID后）
  • 审批下一阶段计划
  • 审批例外计划
  • 给予 ad hoc 指导
  • 确认收益已实现
  • 授权项目收尾

📥 接收（上行）

  • 项目启动请求（SU → DP）
  • 项目启动文件 PID（IP → DP）
  • 要点报告（CS → DP，定期）
  • 例外报告（CS → DP）
  • 阶段竣工报告（SB → DP）
  • 例外计划（SB → DP）
  • 项目收尾建议（CP → DP）

📤 发出（下行授权）

  • 已授权项目启动 → IP
  • 已授权项目交付 → CS（首次）
  • 已授权阶段 → CS（每阶段）
  • 例外计划批准 → CS
  • 提前收尾通知 → CP（例外时）

👥 角色

  • 项目发起人 — 最终决策权
  • 高级用户 — 用户需求代表
  • 高级供应商 — 技术能力代表
  • 项目保证 — 独立监督

📋 项目准备 SU

万事俱备而后动 — 确保项目值得做、做得了、有人管

🔑 关键活动

  • 任命项目发起人和项目经理
  • 捕捉先前项目经验教训
  • 设计和任命项目管理团队
  • 编制项目概述文件
  • 编制启动阶段计划

📦 产出管理产品

  • 项目概述文件 → IP
  • 日志 初建
  • 风险登记单 初建
  • 问题登记单 初建
  • 经验教训记录单 初建

👥 角色

  • 项目经理 — 执行活动
  • 项目发起人 — 批准启动

➡️ 流向

  • → DP：申请授权启动项目
  • → IP：移交项目概述文件

📋 项目启动 IP

奠基立业 — 建立项目基线，获得正式授权

🔑 关键活动

  • 建立质量管理方法
  • 建立风险管理方法
  • 建立沟通管理方法
  • 建立收益管理方法
  • 建立可持续性管理方法
  • 建立问题管理方法
  • 编制项目计划
  • 细化商业论证
  • 编制项目启动文件 PID

📦 产出管理产品

  • 项目启动文件 → DP
  • 项目计划
  • 项目产品描述
  • 质量管理方法
  • 风险管理方法
  • 沟通管理方法
  • 收益管理方法
  • 可持续性管理方法
  • 问题管理方法
  • 质量登记单 初建

📥 接收

  • 项目概述文件（来自SU）
  • DP 的启动授权

👥 角色

  • 项目经理 — 主导执行
  • 项目保证 — 独立审核

➡️ 流向

  • → DP：提交PID申请授权交付
  • → CS：提供阶段计划

📋 阶段边界管理 SB

承前启后 — 评审本阶段成果，规划下一阶段

🔑 关键活动

  • 编制下一阶段计划
  • 更新项目计划
  • 更新商业论证
  • 报告阶段结束
  • 编制例外计划（例外时）
  • 更新各登记单

📦 产出管理产品

  • 阶段计划 → DP审批
  • 阶段竣工报告 → DP
  • 例外计划 → DP（例外时）
  • 项目计划（更新）
  • 商业论证（更新）
  • 风险/问题登记单（更新）
  • 经验教训记录单（更新）

👥 角色

  • 项目经理 — 主导执行
  • 项目保证 — 独立审核
  • 项目发起人 — 批准阶段计划

➡️ 流向

  • → DP：申请下一阶段授权
  • → CS：移交已批准阶段计划
  • → CP：最后阶段结束时

📋 阶段控制 CS

运筹帷幄 — 日常管理：分配工作、监督进展、处理问题

🔑 关键活动

  • 授权工作包（→ MP）
  • 监督工作包进展
  • 接收已完成工作包
  • 审查问题和风险
  • 编制要点报告（→ DP）
  • 上报例外报告（→ DP）
  • 触发阶段边界（→ SB）
  • 触发项目收尾（→ CP）

📦 使用/产出管理产品

  • 工作包描述 → MP
  • 要点报告 → DP（定期）
  • 例外报告 → DP
  • 问题报告
  • 产品登记单 维护
  • 风险/问题登记单（持续更新）
  • 阶段计划（来自SB/IP）

👥 角色

  • 项目经理 — 全权负责
  • 项目保证 — 监督检查
  • 变更委员会 — 变更决策

↕ 双向交互

  • ↓ 向MP：授权工作包
  • ↑ 收MP：检查点报告
  • ↑ 向DP：要点/例外报告

📋 项目收尾 CP

功成身退 — 移交产品、总结经验、正式关闭项目

🔑 关键活动

  • 移交产品给运营方
  • 评估项目绩效
  • 总结经验教训
  • 建议项目收尾（→ DP）
  • 发布项目收尾通知

📦 产出管理产品

  • 项目竣工报告
  • 项目收尾建议 → DP
  • 商业论证（最终评审）
  • 经验教训记录单（最终汇总）
  • 收益管理方法（更新收益计划）

👥 角色

  • 项目经理 — 主导执行
  • 项目发起人 — 批准收尾
  • 高级用户 — 确认产品验收

➡️ 流向

  • → DP：建议收尾，等待最终授权
  • → 运营方：移交项目产品
  • → 组织：沉淀经验教训

📋 产品交付管理 MP

督工之道 — 团队接受工作包、执行工作、交付产品

🔑 关键活动

  • 接受并确认工作包
  • 制定团队计划
  • 执行工作（实际交付）
  • 提交检查点报告（→ CS）
  • 完成质量检查
  • 移交已完成产品
  • 报告工作包完成

📦 使用/产出管理产品

  • 工作包描述（来自CS）
  • 检查点报告 → CS（定期）
  • 产品描述 验收标准
  • 质量登记单（填写测试记录）
  • 经验教训记录单

👥 角色

  • 团队经理 — 带领团队交付
  • 项目保证 — 质量独立审核
  • 项目经理（CS侧）— 授权与接收

↕ 双向交互

  • ↑ 向CS：检查点报告
  • ↑ 向CS：完成通知
  • ↓ 收CS：工作包授权
